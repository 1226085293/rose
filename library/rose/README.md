#### 服务 - service - 在复杂业务场景下用于做业务逻辑封装的一个抽象层
 * 使用场景：保持业务逻辑的独立性，抽象出来的 service 可以被多个模块复用，将逻辑和展现分离。
   * 复杂的模块跳转、验证、登录等操作
   * 周期运行的方法，例如全局的倒计时等功能
   * 复杂数据的处理，例如展现的数据从 store 中获取后需要经过一定的计算处理，才能展示。
   * 更新 store 需要一定量的计算才能更新到 store 中。
 * serviceContainer 服务容器，主要存储所有 service 类
 * service 服务基类
